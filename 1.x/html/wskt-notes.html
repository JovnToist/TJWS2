TJWS Websocket (JSR 356) extension module version 1.0 alpha

Release notes

This is first release of JSR 356 implementation in scope of TJWS. The implementation is provided as TJWS extension module and generally optional. It introduces extra dependency as fast classpath scan library which is used for end points deployment. The library is forked at GitHub. 7Bee scripts are provided in the fork. End points deployment is provided only in scope of .war deployed applications. The limitation can be reconsidered in future. Since websockets use non blocking IO, they work only with SelectorAcceptor acceptor. Error code SC_NOT_IMPLEMENTED (501) will be returned at handshake for other types of acceptors.

Known problems and limitations

  1. No SSL support
  2. No extensions processing and negotiation
  3. No idle timeout yet
  4. Debug messages are not cleaned completely from log

Build Notes

If you prefer build everything yourself, for example for specific Java version, you need also obtain source code of JSR 356 API. Use the command: svn checkout https://svn.java.net/svn/websocket-spec~source-code-repository, 7Bee build scripts for the API can be obtained from examples/jsr356api of 7Bee distribution.
